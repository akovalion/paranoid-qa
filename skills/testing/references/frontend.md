## 2. Frontend / UI

### 2.1 Fields and Forms
- **Input types** matched to purpose (text/email/tel/url/number/password/search/date/file…); mobile virtual keyboard follows `inputmode`/`type`; number — spinners, step, mouse wheel, e/E/+/−/decimal separator; visual type matches the actual type.
- **Input masks/autoformat:** phone (+1 vs local prefix formats, paste with varying separators), date (dd.mm.yyyy, invalid 31.02/13th month), card (grouping, BIN, Luhn, CVV); cursor does not jump to the end when editing mid-value; Backspace across a separator; paste longer than the mask; mask does not break copying.
- **Length limits:** maxlength N−1/N/N+1 (block or truncate), counter matches; minlength; paste longer than the limit (truncate, not reject); emoji/surrogate pairs in the count; 10k+ characters without freezing; frontend vs backend limit.
- **Special chars/unicode/RTL:** `< > & " ' \` / \\` escaped; HTML tags not executed; emoji (ZWJ, flags, modifiers); RTL and mixed LTR+RTL; normalization (NFC/NFD); zero-width; homoglyphs; long words without spaces (word-break).
- **Paste/autofill:** paste with spaces/line breaks/tabs (trim), formatted text (strip formatting), into a masked field; browser autofill lands in the correct fields; **autofill often does not fire the `input` event → field looks "empty" to live validation**.
- **Whitespace/trim/case:** spaces-only in a required field = empty; leading/trailing trim; NBSP/tab/line break in a single-line field; email lowercased, name/password case preserved.
- **Keyboard/focus:** Tab order follows the visual flow; visible focus ring; Enter = submit/next (per design, no double submit); Esc; arrows/Home/End in dropdowns/date pickers; autofocus on the first field, focus returned after a modal; wrong keyboard layout when Latin input is expected.
- **Validation and errors:** timing (live/on-blur/on-submit per spec); don't paint an untouched form red before interaction; error clears once fixed; message specific (not "Error 422"); focus/scroll to the first invalid field; error linked to its field (`aria-describedby`); submit disabled vs always active; double-submit protection (loader/disable); server error maps to the field; data preserved on error.
- **Required/dependent fields:** conditional requiredness; cascading dropdowns (country→cities, dependent reset); mutually exclusive/interdependent (date "from" ≤ "to"); hiding/showing fields does not lose entered data; totals recalculated.
- **Reset/persistence:** "Clear" resets to defaults (not to empty); draft (localStorage/server) restores; beforeunload on unsaved changes; bfcache on Back; resubmission after refresh (POST).
- **disabled vs readonly:** disabled — not editable/not in tab order/not submitted; readonly — in tab order and submitted; states visually distinct; placeholder ≠ value.
- **Dropdowns/multiselect:** native vs custom; search (case-insensitive, partial match); "nothing found"; long list (virtualization/perf at 1000+); chips for selected items, limit N; grouping, disabled options; keyboard selection; positioning near an edge (flip/shift); duplicate/empty options.
- **Counters:** real-time, correct on paste/delete; color change near the limit; count emoji the same way as maxlength.

### 2.2 Visuals and States
- **All states of every interactive element:** rest, hover, focus-visible (keyboard shows the ring, mouse does not), focus, active/pressed, disabled, loading, selected/checked, error/invalid, read-only. Verify hover **empirically** (real hover + computed-style diff), don't trust `:hover` in the stylesheet. On touch — no "stuck" hover. Focus ring not clipped by `overflow:hidden`. Disabled is not clickable/does not submit on Enter, cursor `not-allowed`. Loading: spinner inside the button, repeat click blocked, dimensions don't "jump". Combinations (hover+focus, disabled+loading, error+focus) behave predictably. States inside portals/overlays work the same.
- **Comparison against the design (Figma):** texts character-for-character (case, punctuation, guillemets, NBSP, ellipsis character); icons (set/stroke weight/size/color/filled-outline); sizes/spacing/padding/gap/click target — numerically (DevTools), not by eye; typography (family/size/weight/line-height/letter-spacing/color) per design token; alignment (icon+text baseline, grid); radii/border/shadows numerically. **Discrepancies → question for analyst/designer.**
- **Design-token consistency:** one "black"/"accent" across screens (no #000 vs #0A0A0A); uniform radii/shadows/font/size scale; 4/8-pt spacing grid (no 7/13/18px); identical dimensions for same-type controls; a single hover/focus/disabled pattern.
- **Overflow/wrapping:** long text in a button/chip/heading (wrap vs ellipsis per design); long word/URL does not overflow (`word-break`); line-clamp truncates correctly; placeholder longer than the field; long values in tables (tooltip); dynamic content (empty/long/emoji/RTL); large numbers (999+, 1 000 000); text at 200% zoom.
- **Z-index/portals/overlays:** modal/dropdown/tooltip/toast above everything including the sticky header; correct stacking of multiple overlays; backdrop intercepts clicks, scroll-lock without a "jump"; close via backdrop/Esc/close button + focus returns to the trigger; dropdown near an edge (flip/shift), not clipped inside an `overflow:hidden`/`transform` parent.
- **Transient/loading:** mock a delay (route/throttling) and freeze the state — spinners, skeletons, progress, disabled buttons; skeleton geometry = content (no layout jump); no flicker "empty→skeleton→empty→data"; optimistic updates with rollback on error; long request (progress/cancel/timeout instead of an eternal spinner); parallel loads don't shift the layout.
- **Animations:** duration/easing, no jank or "jump"; does not block interaction; open/close symmetry without clipping content; **`prefers-reduced-motion` disables/simplifies** (often forgotten); no infinite animations while off-screen; reserve space against CLS.
- **Dark theme:** no white flashes (background/inputs/overlays/iframe), no black-on-dark; icons/SVG recolor via `currentColor`; shadows replaced with border/highlight; states distinguishable; on-the-fly switching without flash or reload, persisted, respects `prefers-color-scheme`.
- **Empty states/toasts:** empty (illustration + text + CTA), distinct from error and loading; "no data" vs "nothing found for this filter" — different texts/actions; toasts (position, auto-dismiss, manual close, stacking, don't cover important content, `role=status/alert`); toast type distinguishable by more than color alone; inline vs toast vs modal by purpose.
- **Instrumented checks:** capture computed styles via `browser_evaluate`/DevTools for an exact diff (not eyeballing screenshots); screenshot snapshots of key states with mock data (mask dynamic parts); rendering with fonts not yet loaded (FOUT/FOIT) and images blocked (alt/placeholders); console free of render errors (React keys, hydration mismatch), no 404s for fonts/icons.

### 2.3 Responsive and Cross-Browser
- **Canonical Desktop resolutions:** 1920×1080 (reference), 1536×864 (common laptop, 125% scale), 2560×1440 (2K — stretching, container max-width, side margins); additionally 1366×768, 1440×900.
- **Canonical Mobile:** 414×896 (iPhone 11/XR), 360×800 (average Android), 393×873 (Pixel 7), 430×926 (iPhone 14/15 Pro Max); additionally 320px (minimum, frequent overflow source), 768×1024 (iPad, tablet/desktop boundary).
- **Breakpoints:** render exactly at the boundaries (767/768, 1023/1024, 1279/1280) — no "jumps", clipping, or duplicate desktop+mobile menus; smooth resize 320→1920 (the reflow moment, no overlaps/collapse to 0).
- **Horizontal scroll (critical):** no horizontal scroll on body/containers at any resolution (fixed px widths, long URLs, wide tables, negative margins, `100vw` with a scrollbar, images without `max-width:100%`).
- **Reflow:** multi-column → single column (visual order = DOM, `order` not broken); sidebar moves below/into the burger; menu → burger; tabs → accordion/scroll; hiding labels/columns, table replaced by cards, filters into a bottom sheet; primary buttons full width; sticky CTA does not cover content/safe-area.
- **Touch:** targets ≥44×44 (iOS)/48×48 (Android) with spacing; swipes/sliders/pull-to-refresh/long-press; momentum scroll; hover functionality reachable via tap, no stuck hover.
- **Orientation/keyboard:** portrait↔landscape without clipping; landscape on a short screen does not hide the CTA; virtual keyboard does not cover the field (scroll it into view); `100vh` does not "jump" (use `dvh`/`svh`).
- **Zoom/font scale:** 200% zoom — single column without horizontal scroll, nothing clipped; 400% at 1280px; enlarged system font does not break layout; pinch-to-zoom not blocked (`user-scalable=no` is an antipattern).
- **safe-area/notch:** `env(safe-area-inset-*)`, content not under the notch/home indicator in portrait and landscape.
- **Cross-browser matrix:** Chromium (Chrome/Edge), WebKit (Safari desktop + iOS), Firefox; on iOS all browsers = WebKit. **WebKit specifics:** `position:sticky` in tables, `gap` in flex, `backdrop-filter`, `100vh` (address bar), `date`/`time` inputs, `border-radius`+`overflow:hidden`. **Firefox:** scrollbars, custom checkboxes, number spinners, sub-pixel fonts, `<details>`. **iframe:** scroll/focus on iOS, height on mobile, sandbox.
- **Misc:** `mask-image`/`clip-path`/`mix-blend-mode` (fallback in WebKit/FF); WebP/AVIF fallback, `srcset`/`sizes` per DPR; FOUT/FOIT, `font-display`; sticky header, scroll-snap, scroll-lock; bottom sheets (swipe to close, inner body scroll); Retina 2x/3x (no blur); mobile performance (heavy shadows/blur, 60fps); `prefers-reduced-motion`/`prefers-color-scheme`/`prefers-contrast`/forced-colors.
- **Driving controlled inputs (React/Vue):** set the value via the native setter + dispatch input/change events, otherwise framework state will not update; type masked fields character by character instead of pasting; re-renders invalidate element refs/ids — re-locate elements afterwards.
