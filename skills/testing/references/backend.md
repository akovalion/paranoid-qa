## 3. Backend / Services

### 3.1 API and contracts
- **Methods/endpoints:** correct method for the operation (GET without side effects); `405`+`Allow` on unsupported; OPTIONS/preflight, HEAD; path case, trailing slash, URL-encoding (`%2F`, unicode); nonexistent → `404`, not `500`/HTML.
- **Status codes:** `200` vs `201`(+`Location`) vs `202`(async) vs `204`(truly no body); `301/302/307/308` (method/body preservation), `304`; `400`(validation) vs `422`(semantics); `401` vs `403`; `404` vs `403` (do not disclose existence); `409/410/412/415/429`; `5xx` not on client errors, no stacktrace/SQL/paths in body; no `200` with an error body.
- **Schema/body:** validate request/response against OpenAPI/JSON Schema (types, formats, `required`, `enum`, `pattern`, `min/max`); formats (ISO 8601 with TZ/offset, UUID, email, money — precision/rounding), `null` vs absence vs empty string; numeric bounds (int32/int64/overflow, negatives, fractional where integer expected); nesting/empty `[]` vs `null`/duplicates in array; encodings (UTF-8, emoji, surrogates, null byte).
- **Fields:** missing required → clear error naming the field; wrong type (`"5"` vs `5`, bool as `0/1`); optional with defaults; read-only/computed ignored on write without `400`; sensitive (passwords/tokens/PAN) not returned.
- **Extra/unknown fields:** strict vs lenient strategy is fixed; extra field does not enable mass assignment (`isAdmin`/`roleId`/`price`); typo in name ≠ missing required.
- **Malformed payload:** invalid JSON → `400` not `500`; empty body/`{}`/empty array; duplicate keys; `Content-Type` mismatch with body; deep nesting/cycles (DoS limit); large payload → `413`.
- **Contracts/compatibility:** drift detection (implementation = OpenAPI); consumer-driven (Pact); backward compatibility (do not remove/rename fields, do not narrow types, do not change enum without a version; adding a required field to request = breaking); versioning (URL/header/media-type), `Deprecation`/`Sunset`.
- **Headers:** `Content-Type`+charset; `Accept`/content negotiation, `406`; cache (`Cache-Control`/`ETag`+`If-None-Match`→304/`Vary`); CORS (origin/methods/headers/credentials, wildcard vs credentials); security (HSTS, nosniff, X-Frame-Options, CSP, no `Server`/`X-Powered-By` with versions); correlation (`X-Request-Id` accepted/returned).
- **Pagination/sorting/filter/projection:** offset/limit vs cursor; `limit=0`/negative/over cap, offset beyond range → empty list not an error; stability under insert/delete (duplicates/gaps), consistency of `total`/`hasNext`; sorting by allowed fields, by disallowed → `400`/ignore, `null`-ordering; filters (operators, AND/OR, injection via filter); projection/sparse fieldsets (exclusion of sensitive).
- **Idempotency:** GET/PUT/DELETE idempotent; repeated DELETE → `404`/`204` (fix which); `Idempotency-Key` (repeat=same result; same key+different body=conflict; TTL); concurrent identical (one success, no duplicates); optimistic locking (`If-Match`/version → `409`/`412`); retry after timeout without double effect.
- **PATCH:** Merge Patch (`null`=delete) vs JSON Patch (operations); PATCH of nonexistent; empty patch (no-op); read-only/immutable; does not wipe fields not passed; validate the resulting object as a whole.
- **Edge:** duplicate query (`?id=1&id=2`), arrays in query; `null`/`""`/absence/`0`/`false` in semantics; unicode normalization in id/search; ID > 2^53 as string for JS; TZ/DST in date filters; double slash/double URL-encoding/`414`; empty collection → `200`+`[]` not `404`; gzip/br/decompression bomb; chunked without `Content-Length`; break mid-stream.

### 3.2 Data and DB
- **CRUD persistence:** written = returned = actually in DB (verify with a SELECT, not just the API response); Read returns the stored value without service fields; Update changes only what was specified (`updated_at` updated, `created_at` not); Delete truly deleted/flagged, repeat idempotent, no "zombies" in related; column defaults; auto-fields (id/created_at/version); NULL vs `""` vs absent key are not lost.
- **Integrity:** NOT NULL → error not a "silent" default; UNIQUE (composite, case, trailing-spaces, NULL in UNIQUE); FK (nonexistent parent → reject, deleting a parent with children); CHECK/domain at the DB level; cascades ON DELETE/UPDATE (no unexpected mass delete); partial unique (only for active).
- **Transactions:** atomicity (mid-failure → full rollback); rollback on exception/constraint/timeout; savepoints; side-effects outside DB (email, event) not on rollback (transactional outbox); long transactions (locks, WAL growth).
- **Isolation/concurrency:** levels (READ COMMITTED/REPEATABLE READ/SERIALIZABLE) actual vs expected; lost update; dirty/non-repeatable/phantom read; check-then-insert race → duplicates (UNIQUE+upsert/`ON CONFLICT`); deadlock (retry, no hang); optimistic (version→409) vs pessimistic (SELECT FOR UPDATE); concurrent decrement of stock/balance (no negative/double debit).
- **Migrations:** forward on a clean and on a populated DB (without data loss); backward/rollback; idempotent/repeated run; compatibility with old code (expand/contract, nullable column before backfill); backfill in batches without locking; index CONCURRENTLY; schema drift (real DB = migrations, no manual edits).
- **Cache/service consistency:** cache invalidation on write (no stale), TTL, cache stampede; dual-write (DB + event atomically via outbox/CDC); eventual consistency (read-after-write for one's own request); denormalized projections are updated; saga/compensations.
- **Soft-delete:** deleted not in queries/search/aggregates; UNIQUE accounting for soft-delete (can create with the same key after deletion); FK/joins on soft-deleted; restore; no "half-deleted".
- **Uniqueness/duplicates:** normalization before comparison (trim/lowercase/unicode email/phone); idempotency by business key; dedup on import/batch; race before commit.
- **Indexes/plan:** is the index used (EXPLAIN/ANALYZE, no seq scan on large); degradation as data grows, queries without LIMIT, N+1 to DB; keyset instead of offset on deep pages; index impact on write/bloat.
- **Encodings/collations:** UTF-8 across the whole chain (utf8mb4 for emoji); collation/case in comparison/sorting/uniqueness; truncation bytes vs characters (do not split a multibyte).
- **Numbers/money:** DECIMAL/integer-minor not float; rounding (half-up/banker's) uniform; type overflow; scale DECIMAL(p,s); split without losing cents.
- **Time/TZ:** storage in UTC (timestamptz), single standard; conversion on read; DST; sub-second precision; range boundaries (inclusive/exclusive); DB vs application time divergence.
- **Retention/GDPR:** deletion/anonymization by term (actually erased where the law requires); right to be forgotten from all places (main DB, replicas, cache, backups, logs, analytics, index); archival to cold; anonymization cascade; deletion audit.
- **Seeding/fixtures:** test isolation (clean state, no dependency on order); cleanup strategy (rollback/truncate, sequence reset); fixtures match the schema; data uniqueness across parallel workers; determinism (fixed seed/time/random).

### 3.3 AuthN/AuthZ and multitenancy
- **AuthN schemes:** **JWT** (alg=none, RS256→HS256 with the public key, weak HMAC; verify `iss`/`aud`/`exp`/`nbf`/`iat`; `kid` manipulation — traversal/SQLi/SSRF on JWKS); **OAuth2/OIDC** (`state`/CSRF, `nonce`/replay, PKCE; `redirect_uri` substitution, code reuse, mix-up); **API-key** (header vs query/leak to logs, rotation/revocation, binding to scope/tenant); **Basic** (TLS only, brute-force, comparison timing); **mTLS** (validity/CA chain/revocation, CN/SAN binding, downgrade).
- **Token lifecycle:** access not "eternal" (expired → 401 not 200); refresh rotation + reuse-detection (stolen refresh → family invalidation); revocation/blacklist (logout truly invalidates on the server, not just client); password change/lockout invalidates sessions; clock skew between services.
- **AuthZ:** check on **EVERY** endpoint, not only UI/gateway (defense in depth); RBAC (no role, multiple roles, role removal within a session); ABAC/scope (insufficient → 403, read does not grant write); 401(not authenticated) vs 403(no rights); method-specific (GET yes, POST/PUT/DELETE no; HEAD/OPTIONS do not bypass); authorization on nested (`/orders/{id}/items` — both order and item); default-deny.
- **Escalation:** vertical (admin endpoint under a user, `role`/`isAdmin` in PATCH, mass-assignment); horizontal (another's resource at the same level); via password/email change of another; privilege persistence (role downgrade does not revoke the token).
- **IDOR/BOLA:** access to another's object by `id` (sequential/predictable) on read/modify/delete; UUID ≠ authorization; IDOR in path/query/body/headers/cookie/nested JSON; IDOR on write (`owner_id`/`tenant_id` of another); in exports/reports/webhooks/files; indirect (filters/batch return another's); BOPLA (object properties not for this role).
- **Multitenancy:** full isolation A↔B; tenant from the **token**, not from a client parameter (`X-Tenant-Id` is spoofable); tenant conflict token ≠ URL → reject; WHERE tenant_id in all queries (leak via JOIN/aggregates/reports); shared cache with tenant in the key; cross-tenant via related entities/webhooks/jobs/export; file/bucket isolation; search does not cross boundaries.
- **Negative token states:** absent → 401; expired → 401; forged/corrupted (broken base64, extra segments, `Bearer ` without value) → 401; wrong scheme/case; token in query instead of header; token from another environment/audience; very long (DoS).
- **Replay/CSRF/brute-force:** replay of a valid request (payments) — nonce/idempotency-key; token replay from another IP/UA (binding); CSRF on cookie sessions (token/`SameSite`+`Origin`/`Referer`, override via `_method`); rate-limit on login/refresh/reset per account AND per IP, lockout without DoS, protection against `X-Forwarded-For` spoofing; account enumeration (identical response/timing); OTP/2FA limit.
- **Forgotten endpoints:** without authorization at all (admin/actuator/metrics/health with PII, swagger, GraphQL introspection, `/internal/*`); technical (export/import/bulk/webhooks-receiver); versioned duplicates (`/v1` closed, legacy open); direct access bypassing the gateway; token/PII leaks in logs/errors; gRPC/queues/GraphQL mutations — the same authorization.

### 3.4 Async / queues / events / jobs
- **Producer:** message after the business transaction commit (not before/not phantom); correct topic/routing/partition key; schema (Avro/Protobuf/JSON Schema, version); headers (correlation/trace/message-id/timestamp); behavior when the broker is unavailable; publisher confirms/acks/nack; size > broker limit (claim-check).
- **Consumer:** processing exactly once logically; ack only after success (behavior on crash before ack); manual vs auto-ack, prefetch; invalid payload does not crash the consumer; unknown type/version → graceful skip or DLQ; processing longer than visibility timeout → redelivery/duplicate.
- **Guarantees/idempotency:** at-least-once → consumer is idempotent (dedup by message-id/business-key, upsert); exactly-once (what backs it — outbox/Kafka EOS, not marketing); redelivery does not create duplicates in DB/side-effects (emails/payments); deduplication window (TTL) and behavior after.
- **Ordering:** globally/within partition/within key; one business-key into one partition; out-of-order (created before updated, delete before create); rebalance/change in partition count; concurrency>1 breaks ordering — is that acceptable; version/sequence to discard stale.
- **Retries/DLQ:** only transient are retried (business errors straight to DLQ); exponential+jitter, max attempts; counter preserved across retries; in DLQ — payload+reason+stacktrace+attempt; redrive from DLQ without re-failing; DLQ is monitored/alerts; retry does not duplicate side-effects.
- **Poison messages:** an always-failing one does not block the partition indefinitely; no infinite retry loop; isolation in DLQ; head-of-line blocking.
- **Duplicates:** from producer/consumer retries, rebalance, redelivery after crash; broker dedup vs application; double processing of a webhook/payment; multiple consumer instances.
- **Eventual consistency:** convergence time (read-after-write may return stale); test — polling until a condition, not a fixed `sleep`; lag SLA; replica lags; partial convergence (one projection updated, another not).
- **Sagas/outbox:** happy path of the whole chain; failure at each step → compensations of the previous; compensation is idempotent; compensation impossible (email sent, money withdrawn); step timeout (no eternal in-progress); double start by business-key; outbox in the same transaction, does not publish on rollback, order=records, table cleanup; inbox for dedup of incoming.
- **Webhooks:** delivery (2xx=success); signature (HMAC) verified; replay protection (timestamp+nonce, window); retries on 5xx/timeout + backoff + DLQ; receipt idempotency (event-id not twice); response timeout on the endpoint; out-of-order resilience; duplicates (dedup mandatory); unavailable/stale URL; mTLS/IP allowlist; payload versioning.
- **Schedulers/cron:** run at the appointed time (cron expression); TZ/DST (skip or double run at 02:00–03:00); missed run (catch-up vs skip); overlap (lock/skip/queue); single-execution in a cluster (distributed lock/leader election); failure (alert/retry); idempotency; a long job does not run past the window.
- **Long-running:** progress (queued/running/done/failed, %/stages); cancellation in progress (actual stop, resource release); timeout (killed, does not hang); worker restart (checkpoint/safe restart); partial result; resubmit of the same job-id.
- **Loss/delay:** message loss is detected (auto-ack before processing); durability/replication; growing consumer lag (metric/alert); backpressure (producer faster than consumer); message TTL; queue overflow (reject/eviction).
- **Tooling:** end-to-end tracing (correlation through producer→broker→consumer→side effects); metrics (lag, DLQ size, retry rate); test waits via polling; isolation (unique topics/queues per test run, cleanup of offset/queue/DLQ); fake clock for schedulers; ability to force a duplicate/reordering/consumer crash in the test.

### 3.5 Non-functional and resilience
- **Load:** load (SLA at target RPS, latency p50/p95/p99/p999, error rate, CPU/RAM/IO saturation); stress (knee point, degradation without crash, 429/503 instead of 5xx); soak/endurance (memory/connection/descriptor leaks, GC pause growth); spike (x10 growth, recovery speed). Metrics — **percentile, not average**; high-percentile tails; coordinated omission (open vs closed model); warm-up before measurement; realistic request mix; think-time.
- **Scalability:** horizontal scaling actually raises throughput; stateless (any instance — any request); sharding (no hot partition/key); autoscale (threshold, speed, no flapping, thundering herd); bottleneck (DB pool/locks, cache, external API); balancer (uniformity).
- **Degradation/limits:** back-pressure vs uncontrolled crash; limits (max request/response size, max connections/queue, pool exhaustion); rate limiting (counters, fixed/sliding window, 429+Retry-After, isolation per client/tenant); quota; prioritization (important traffic not crowded out, load shedding).
- **Resilience:** timeouts on all calls (connect/read/write/total, cascade ≤ parent's); retries only idempotent (backoff+jitter, no retry storm, budget); circuit breaker (open on threshold, half-open, recovery); bulkhead (pool isolation); backpressure; **edge:** retry on top of timeout → duplicates (needs idempotency); nested retries at every layer (retry amplification); client timeout < server (server does useless work).
- **Idempotency under repeat:** idempotency-key (repeat=prior result); duplicates from at-least-once queues (exactly-once on effect); repeat after timeout (no double debit); concurrent repeat of a key (race); key TTL; partial failure (applied, response did not arrive → repeat completes correctly).
- **Dependency failure:** DB unavailable (error vs hang, readiness pulls out of rotation); cache unavailable (fallback to source, protection against stampede); third-party API (timeout, fallback, feature degradation); queue/broker; **partial degradation** (a slow dependency — the trickiest case); graceful degradation (disable secondary, keep the core); **edge:** DNS failure, TLS-handshake fail, expired certificate, 200 with a broken body, a flapping dependency.
- **Chaos / fault injection:** kill -9 of an instance (no data loss/stuck transactions); network failures (packet loss, partition/split-brain); resource degradation (CPU throttle, disk fill, OOM, inode); latency/error injection on % of traffic; clock skew/NTP jump; game day/runbook check; simultaneous failure of several dependencies; leader failure (failover).
- **Recovery:** restart (state recovery, drain in-flight); after a crash (unfinished transactions roll back/roll forward, no "orphans"); recovery after a dependency returns (pool reconnect); cold start (start order, no deadlock); reconciliation of deferred events; RTO/RPO meet targets; **edge:** restart under load; simultaneous restart of all instances; leader loss during a long operation.
- **Resource leaks:** memory (heap without release under soak, native, cache without TTL); connection pool (release on exception, idle timeouts); file descriptors/sockets (close after error, TIME_WAIT); threads/goroutines (hung workers, thread pool exhaustion); unbounded queue → OOM; temp files/log rotation; **edge:** leak only on the error path / on client request cancellation / slow (visible over hours).
- **Observability (as a testable property):** structured logs (JSON, levels, no PII/secrets, context, logging on every catch); RED metrics (Rate/Errors/Duration) + USE (Utilization/Saturation/Errors), label cardinality; distributed tracing (span per call, propagation through services and queues); correlation-id (generated at entry, propagated to all downstream and logs, returned to the client); alerts on symptoms (SLO burn rate), a runbook for each, verify it actually fires; **edge:** logs under load are not a bottleneck, trace loss through a queue.
- **Health probes:** liveness (is the process alive, independent of external dependencies — otherwise cascading restart); readiness (dependency readiness, pulls out of rotation on degradation); startup probe for slow start; **edge:** readiness on an optional dependency (false removal from rotation); "green" health while the service does not serve (shallow probe); flapping readiness.
- **Config/secrets/flags:** config validation at start (fail-fast, typing, precedence CLI>ENV>file>default); environment separation, no prod secrets in repo/image/logs; secret rotation without restart (validity window of two keys); config hot reload (atomicity); feature flags (both branches, default when the flag service is unavailable — fail-secure, sticky bucketing, kill-switch instant, flag not the sole authorization barrier, no desync between replicas).
- **Zero-downtime deploy:** rolling update (old+new versions simultaneously, contract compatible N and N+1); schema migration expand-contract (compatible with old code, reversible); message contract (tolerant reader, unknown fields ignored); graceful drain (SIGTERM → complete in-flight, terminationGracePeriod); fast rollback without data loss (irreversible migrations isolated); canary/blue-green (traffic share, auto-rollback on metrics); **edge:** removing a field before clients update; DB and code rollout order; a long-running request crosses the deploy; in-flight message in the old format; mixed version cluster longer than expected.
- **Containers/k8s:** liveness/readiness/startup; graceful shutdown (drain); stateless in N replicas (no local state/files); resource limits (OOMKill, CPU throttling); rolling/canary; config/secrets via ConfigMap/Secret; DNS resolve/timeouts inside the cluster; persistent volumes; HPA triggers without crash loop; container TZ/locale (UTC), CA in the image; non-root, read-only FS.

### 3.6 Security backend (OWASP API Top 10)
- **1. Injection:** SQL/NoSQL in every parameter/header/cookie (`' OR '1'='1`, `$ne`/`$where`); blind (time-based `SLEEP`, boolean, OOB DNS/HTTP); second-order (fires on read); command/OS (`; | $(cmd) &&`); LDAP; XPath/XXE/SSTI (`{{7*7}}`/`${...}`); ORM/GraphQL; **edge:** ORDER BY/LIMIT/column names (not parameterized), LIKE masks, encoded payloads, numeric context without quotes.
- **2. BOLA/IDOR:** substituting `id`/`uuid`/`slug` in path/query/body → 403/404 not 200; enumeration of sequential and leaked UUIDs; nested resources; write/delete of another's (not just read); via batch/GraphQL node/export; **edge:** indirect ownership (order→payment→document) — check at every level of the chain.
- **3. Broken Authentication:** brute-force login/OTP without lockout/rate-limit, credential stuffing; weak password, no check against leaked; JWT (`alg:none`, RS256→HS256, weak secret, no exp/iss/aud check); no invalidation on logout/password change; user enumeration (different errors/timings); password reset (predictable/long-lived token, host-header injection); **edge:** OTP without TTL/limit, 2FA bypass at the second step.
- **4. BOPLA:** mass assignment (`role`/`isAdmin`/`balance`/`status`/`price` in body on create/update → ignore; via nested/arrays/alias/PATCH); excessive data exposure (`passwordHash`/`internalNotes`/another's email/tokens in the response — filtering on the server, not client; leak via `include`/`expand`/`fields`/error objects/debug); **edge:** different roles — different field set (role×field matrix), list returns more than detail.
- **5. BFLA:** admin endpoints under a user (`/admin/*`, `/internal/*`, DELETE/PUT); method override (`X-HTTP-Method-Override`); functions of another role horizontally; hidden/undocumented endpoints (from JS/swagger/old versions); **edge:** authorization only on UI/gateway, not on the service; alternative route (gRPC/internal).
- **6. Unrestricted Resource Consumption:** no rate-limit on login/OTP/search/export/registration/email-SMS; limit bypass (`X-Forwarded-For`, UA, email case, `+alias`); expensive operations (large `limit`, recursive GraphQL, batch without a limit); no limits on body/file size/number of fields (DoS, billing abuse); **edge:** ZIP bomb, ReDoS (catastrophic regex), CPU/memory complexity.
- **7. SSRF:** any URL parameter (webhook/callback/import-by-url/avatar/preview) → `http://169.254.169.254/` (cloud metadata), localhost, internal-IP, `file://`/`gopher://`; filter bypass (DNS rebinding, redirect to internal, IP in decimal/octal/hex, `[::1]`, `http://allowed@internal`); blind SSRF (timeout/OOB); **edge:** via SVG/XML/PDF parsing, via webhook validation.
- **Deserialization:** unsafe (Java gadget chains, Python pickle, PHP unserialize, YAML `!!python/object` → RCE); polymorphic (Jackson `@type`); **edge:** from cookie/cache/queue/Redis.
- **8. Security Misconfiguration:** headers (no HSTS/nosniff/CSP/X-Frame-Options, no `Cache-Control:no-store` on sensitive, `Server`/`X-Powered-By` present); CORS (`*`+credentials, Origin reflection, `null`-origin, regexp bypass, subdomain holes); TLS (HTTP→HTTPS redirect, weak ciphers TLS1.0/1.1, certificate validity, mixed content); at-rest encryption (PII/tokens in cleartext, unhashed passwords or MD5/SHA1 without salt); default creds, open actuator/`/debug`/`/env`/swagger in prod, directory listing, verbose errors.
- **9. Improper Inventory Management:** live deprecated versions (`/v1`, beta, staging with prod data); undocumented/debug endpoints, forgotten test accounts; mismatch of implementation and swagger; **edge:** the same data through several APIs with different protection.
- **10. Unsafe Consumption of 3rd-party:** trusting external API responses without validation (injection/SSRF via the integrator); no TLS/signature check; dependencies with CVEs (`npm audit`/`pip-audit`/Trivy/Snyk, SCA in CI); **edge:** a vulnerable dependency in one microservice, lock-file does not reflect the real versions.
- **Cross-functional:** input validation/sanitization (type/format/range/length/enum, null-byte `%00`, CRLF into headers/logs, canonicalization before validation, path traversal `../`/`..%2f`); file upload (magic bytes not extension, double-extension `.jpg.php`, limit before and after unpacking, ZIP bomb, antivirus/EICAR, storage outside webroot, name randomization, SVG/polyglot with XSS/SSRF); secrets/leaks (in code/git history/Docker layers — trufflehog/gitleaks; in logs/responses/error/URL/JWT payload); audit logs (login/logout, privilege change, PII access, financial, authorization failures — who/what/when/from where, immutability); races/TOCTOU (double debit, limit bypass, coupon reuse); cache poisoning; 401/403/404 do not disclose existence.
