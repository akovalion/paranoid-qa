## 4. Сквозное (оба трека)

- **Негатив/ошибки/сеть (моки).** Покрыть каждую ветку статуса: 400(валидация→поля)/401(редирект на логин, return URL)/403/404/409/422/429(`Retry-After`)/500/502/503/504 — понятное сообщение без stacktrace, без `undefined`/`[object Object]`, без двойного отображения (тост+inline). Тело не по контракту (HTML вместо JSON, обрезанный JSON, пустое при 200). Таймаут (не вечный спиннер). Медленная сеть (скелетоны, дизейбл). Offline в момент сабмита (данные не теряются), восстановление, флаппинг (нет каскада дублей). Retry (только ретраибельные, backoff, идемпотентность). Деградация без падения UI (error boundary, плейсхолдеры, сломанная картинка → alt). **Playwright-моки:** `page.route` + `route.fulfill({status})` для каждого ветвления; `route.abort('failed')`; искусственная задержка → лоадеры; битое тело `route.fulfill({body:'<<not json'})`; `context.setOffline(true)`; гонки (задержать первый, отпустить второй); последовательные ответы (счётчик: 500→200 для retry); инспекция `request.postData()`/заголовков (idempotency-key, Authorization, дубли); CDP `Network.emulateNetworkConditions` для реалистичной медленной сети; снять роуты после теста.
- **Интеграции (запрос/ответ/payload).** Для каждого действия зафиксировать HTTP (метод/URL/query/headers/body); статус соответствует семантике; payload по контракту (имена/типы/вложенность); UI рендерит из ответа, не из стейта/мока; одно действие = один запрос (нет дублей/N+1); нет висящих запросов после ухода (AbortController). Payload: обязательные заполнены реальными значениями; нет лишних/debug-полей; чувствительные не в query/в открытом виде; clientIp/fingerprint не `undefined`/`null`; маскированные на UI уходят в полном формате; деньги в правильной единице (рубли vs копейки, нет потери float); даты в TZ/ISO без смещения ±1 день; bool/enum в формате бэка (`true` vs `"true"`); кодировка спецсимволов. **Consistency UI↔Backend:** показанное после операции = реально сохранённое (повторный GET); optimistic update откатывается при 4xx/5xx; агрегаты/счётчики совпадают с пересчётом; статусы не хардкод «Успешно»; данные доступны на других экранах (нет рассинхрона кэшей).
- **State/сессия.** Логин создаёт сессию (cookie HttpOnly/Secure/SameSite); логаут реально инвалидирует на сервере (Back из bfcache не отдаёт приватное); таймаут по бездействию (sliding vs абсолютный); refresh-токен (автопродление, протухший → логин); параллельные сессии; «запомнить меня»; «выйти со всех устройств»; F5 на любой странице (восстановление/сброс без белого экрана); перезагрузка во время in-flight (нет дубля POST); beforeunload при несохранённом; смена/понижение роли в активной сессии (права обновляются/403 на запретный URL); деактивация обрывает сессию; мультиаккаунт не смешивает контексты.
- **localStorage/sessionStorage.** sessionStorage изолирован по вкладкам, localStorage общий; чувствительные данные не в открытом виде; порча/удаление ключей не роняет (невалидный JSON → фолбэк); QuotaExceeded; очистка/приватный режим (мягкая деградация); версионирование схемы (старый формат мигрируется/сбрасывается); логаут чистит пользовательские данные.
- **Мультивкладочность.** Логаут в одной → реакция в других (storage event); логин подхватывается; изменение данных в A отражается в B; параллельное редактирование одной сущности (optimistic lock, нет lost update); Ctrl+click сохраняет контекст.
- **Навигация/роутинг/404/deeplink.** Back/Forward восстанавливают состояние (фильтры/пагинация/табы/скролл, URL=контент); SPA History API без рассинхрона; Back после сабмита не дублирует; deep-link на сущность/таб/фильтр открывает напрямую; защищённая страница без авторизации → логин + возврат; несуществующий/удалённый ресурс → 404/empty не белый экран; URL-параметры состояния переживают шаринг; ручная инъекция невалидных query → дефолты; хэш/якоря скроллят; trailing slash/регистр/редиректы (301/302, петли); `target=_blank`+`rel=noopener`.
- **Время/локаль/i18n/TZ.** Даты в TZ пользователя vs сервера; смена TZ устройства; DST (несуществующий/дублирующийся час); граница суток/недели/месяца/года (фильтр «сегодня» учитывает TZ); clock skew (токены/«5 минут назад» не в будущее); 29.02/високос; полночь UTC vs локальная (нет «съезда» на день); очень старые/будущие даты. Переключение локали меняет все строки (нет raw i18n-key); форматы даты/времени/чисел/валюты по локали; plural rules; длинные переводы не ломают вёрстку; RTL-зеркалирование; интерполяция переменных; локаль влияет на сортировку (collation) и регистр.
- **Нефункциональное/перформанс/консоль.** Консоль открыта весь прогон: отличать `error`/`warning`, снять baseline шума стенда (CDN/аналитика/расширения), отличить шум от бага (только при действии с тестируемым, домен приложения, по таймстемпу); `Uncaught TypeError ... undefined` после API = незащищённый рендер; React keys/hydration mismatch; CSP/CORS/mixed content; повторить в инкогнито без расширений. Core Web Vitals (LCP<2.5s, CLS<0.1, INP/TBT) с мобильной эмуляцией + CPU 4x + Slow 4G; CLS-ловушки (картинки/iframe без `width/height`, поздний шрифт, cookie-баннер сверху); тяжёлые ресурсы (несжатые картинки, нет lazy); бандлы (размер, code-splitting); gzip/brotli; кэш статики; водопад/N+1 запросов; виртуализация длинных списков; анимации через transform/opacity (не width/top). Утечки памяти: открыл-закрыл модалку/роут 20–50 раз → heap snapshot (detached DOM, незачищенные setInterval/таймеры/listeners/WebSocket); долгая сессия 30–60 мин (heap не растёт линейно); фоновая вкладка (throttling таймеров).
- **Security-сквозное (с фронта).** XSS в каждом поле (`<script>alert(1)</script>`, `<img src=x onerror=>`, `"><svg onload=>`) — сохранить, перезагрузить, проверить во ВСЕХ местах вывода (шапка, список, превью, PDF, title); HTML-инъекция без скрипта; атрибутный контекст (`javascript:` в href); rich-text/markdown санитизация; DOM-based XSS из URL/хэша; SVG/double-extension при загрузке; template injection (`{{7*7}}`). Транспорт/заголовки: чувствительное не в URL (логи/Referer); `console.log` без токенов/PII в проде; токены в localStorage (XSS) vs httpOnly-cookie; cookie-флаги; весь трафик HTTPS, нет mixed content; HSTS; security-заголовки; CORS не `*` с credentials. Доступ/IDOR/CSRF: прямая ссылка на приватное в инкогнито → логин; доступ после логаута (Back/replay); IDOR (поменять `id`/`userId` на чужой); вертикальная привилегия (admin-эндпоинт под юзером — 403, не скрытая кнопка); «скрытая» защита только в UI (replay запроса из Network); прямые ссылки на файлы/экспорт; CSRF (анти-токен/SameSite); подмена параметров (`price`/`discount`/`isAdmin` в теле); mass assignment; брутфорс/рейт-лимит; user enumeration; открытый редирект (`?redirect=`/`?next=` на внешний домен); защита от повторной отправки платежа.
- **Аналитика/события метрик.** Отправка событий (клики/просмотры/конверсии) с верным payload; нет дублей; события при ошибках; GDPR-режим (только после согласия); работа при блокировщике трекеров (нет падений); не утекают PII/токены в third-party.
- **Файлы/загрузки/экспорт.** Загрузка: форматы по extension И MIME/сигнатуре, подмена расширения; лимиты (0 байт, ровно лимит, +1, гигантский); множественная, дубли имён, кириллица/эмодзи/длинное имя, путь `../`; прерывание/возобновление/прогресс/отмена; drag&drop vs кнопка vs Ctrl+V; битый/polyglot/zip-bomb; превью, удаление/замена. Скачивание: имя/расширение/контент, докачка, параллельные. Печать (Ctrl+P): print-стили, разрывы, скрытие лишнего. Экспорт CSV/XLSX/PDF: соответствие данным на экране, UTF-8 BOM, разделители, **CSV-инъекции (`=`/`+`/`-`/`@` в начале ячейки)**, пустой/большой набор, с фильтрами, локаль чисел/дат.
- **Поиск/фильтры/сортировка.** Поиск: пустой/1 символ/частичное/регистронезависимость/опечатки/транслит; спецсимволы и операторы, пробелы, длинный запрос; подсветка, debounce, отмена устаревших (race); empty state «ничего не найдено», сброс. Фильтры: комбинации, сброс всех, сохранение в URL, конфликтующие, счётчик. Сортировка: по каждому полю, asc/desc, стабильность, с null/пустыми, сохранение при пагинации.
- **Drag&drop/интерактив.** Валидные/невалидные зоны, отмена (Esc), drop вне зоны; сортировка списка + сохранение после релоада; тач-аналог + автоскролл к краю; ресайз панелей/колонок + сохранение.
- **Undo/redo/обратимость.** Ctrl+Z/Y, глубина истории; snackbar undo для «удалить»; массовые операции и откат; автосохранение vs ручное, индикатор несохранённого, потеря при уходе.
- **Скролл/пагинация/виртуализация.** Бесконечный скролл (подгрузка, нет дублей/пропусков на стыке, конец списка); виртуализация при быстром скролле + сохранение позиции при возврате (scroll restoration на Back); пагинация (первая/последняя, прямой URL к странице, размер); sticky-шапка/«наверх».
- **Feature interaction/регрессия.** Взаимодействие новой фичи со смежными (одновременное включение нескольких); регресс затронутых областей; фиче-флаги (вкл/выкл/частичный rollout/A/B — стабильная ветка для юзера); конфликты состояний (две фичи в один стор/URL/localStorage); обратная совместимость со старыми данными/ссылками/закладками; миграции (старые записи после деплоя, дефолты новых полей, старая структура в Storage → upgrade/сброс, рассинхрон версий клиент↔бэк, кэш SW после релиза).
- **Согласия/правовые тексты.** Cookie-баннер (принять/отклонить/настроить, запоминание, нет трекинга до согласия); чекбоксы согласий (оферта/ПДн/рассылка — обязательность, ссылки на документы, нельзя отправить без обязательных); возрастные/региональные ограничения, версия принятых условий, повторный запрос при изменении.
- **Капча/rate-limit (UI).** Появление капчи по триггеру (N неудачных попыток), прохождение, ошибка/перезагрузка капчи, блокировка отправки до прохождения, доступность, поведение при недоступности провайдера; рейт-лимит (логин/код/поиск — достижение лимита, сообщение, кулдаун-таймер, 429 + backoff); защита от дабл-клика (идемпотентность, блокировка кнопки).
- **Платежи и деньги (если применимо).** Округление/копейки/отрицательные/нулевые/лимиты/валюта и смена; 3DS-редирект на банк и возврат, таймаут оплаты, двойное списание, отмена/возврат; промокоды (применение/истечение/комбинирование/невалидный код); OTP/SMS (запрос кода, таймер повтора, лимит попыток, истечение, неверный код, код не в ответе/логах); 3-D Secure (прохождение/отказ/таймаут, финальный статус заказа); PCI DSS (нет данных карты на своём бэке, токенизация).
- **Вебхуки/коллбэки (с фронта).** Статус по вебхуку/поллингу доезжает до UI без ручного refresh; поллинг с лимитом/backoff; поздний/дублированный вебхук не ломает статус; редирект после внешнего коллбэка (3DS/OAuth) на правильную страницу; потеря коллбэка (закрыл вкладку) — операция доводится, статус виден при возврате.
- **Прерывания/устойчивость окружения.** visibilitychange (сворачивание/возврат вкладки), пробуждение из сна, потеря фокуса; поворот экрана во время операции, входящий звонок; обрыв в середине wizard (возобновление с нужного шага); работа при отключённых cookies/localStorage/JS, инкогнито.
- **Email/уведомления.** Триггер письма по событию, единичность (нет дублей), задержка; содержимое (данные, рабочие ссылки, срок жизни токена, unsubscribe); локализация по языку, From/Reply-To, тема; рендер в Gmail/Outlook/мобильном, plain-text fallback; push/in-app/toast (появление, авто-скрытие, стекинг, разрешения, отключение); колокольчик (счётчик непрочитанных, «прочитано», синхронизация вкладок).
