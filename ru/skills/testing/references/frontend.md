## 2. Frontend / UI

### 2.1 Поля и формы
- **Типы input** по назначению (text/email/tel/url/number/password/search/date/file…); виртуальная клавиатура на мобиле под `inputmode`/`type`; number — спиннеры, step, колесо мыши, e/E/+/−/разделитель; совпадение визуального типа с реальным.
- **Маски/автоформат:** телефон (+7 / 8, вставка с разными разделителями), дата (дд.мм.гггг, невалидные 31.02/13-й месяц), карта (группировка, BIN, Luhn, CVV); курсор при правке в середине не прыгает в конец; Backspace через разделитель; вставка длиннее маски; маска не ломает копирование.
- **Лимиты длины:** maxlength N−1/N/N+1 (блок или обрезка), счётчик соответствует; minlength; вставка длиннее лимита (обрезка, не отказ); эмодзи/суррогатные пары в подсчёте; 10k+ символов без фриза; лимит фронт vs бэк.
- **Спецсимволы/юникод/RTL:** `< > & " ' \` / \\` экранируются; HTML-теги не исполняются; эмодзи (ZWJ, флаги, модификаторы); RTL и смешанный LTR+RTL; нормализация (NFC/NFD); zero-width; гомоглифы; длинные слова без пробелов (word-break).
- **Paste/autofill:** вставка с пробелами/переносами/табами (trim), форматированного текста (снятие формата), в поле с маской; браузерный autofill в правильные поля; **autofill часто не триггерит `input`-событие → поле «пустое» для live-валидации**.
- **Пробелы/trim/регистр:** только пробелы в обязательном = пустое; ведущие/хвостовые trim; NBSP/таб/перенос в однострочном; lowercase email, сохранение регистра имени/пароля.
- **Клавиатура/фокус:** Tab-порядок по визуальному потоку; видимый focus-ring; Enter = submit/переход (по дизайну, без двойного сабмита); Esc; стрелки/Home/End в дропдаунах/датапикерах; автофокус на первом поле, возврат фокуса после модалки; раскладка (русская при ожидании латиницы).
- **Валидация и ошибки:** момент (live/on-blur/on-submit по ТЗ); не краснить пустую форму до взаимодействия; ошибка исчезает при исправлении; текст конкретный (не «Error 422»); фокус/скролл к первому невалидному; ошибка связана с полем (`aria-describedby`); submit disabled vs всегда активна; защита от дабл-сабмита (лоадер/дизейбл); серверная ошибка мапится на поле; данные сохраняются при ошибке.
- **Обязательность/зависимые:** условная обязательность; каскадные дропдауны (страна→города, сброс зависимого); взаимоисключающие/взаимозависимые (дата «от» ≤ «до»); скрытие/показ полей не теряет введённое; пересчёт итогов.
- **Сброс/персист:** «Очистить» к дефолтам (не к пустым); черновик (localStorage/сервер) восстанавливается; beforeunload при несохранённом; bfcache при Back; повторная отправка после refresh (POST).
- **disabled vs readonly:** disabled — не редактируется/не в табе/не отправляется; readonly — в табе и отправляется; визуальное отличие состояний; placeholder ≠ значение.
- **Дропдауны/мультиселект:** нативный vs кастомный; поиск (регистронезависимый, частичный); «ничего не найдено»; длинный список (виртуализация/перф 1000+); чипсы выбранных, лимит N; группировка, disabled-опции; выбор клавиатурой; позиционирование у края (flip/shift); дубли/пустые опции.
- **Счётчики:** реальное время, корректны при вставке/удалении; смена цвета у лимита; считают эмодзи как maxlength.

### 2.2 Визуал и состояния
- **Все состояния каждого интерактивного элемента:** rest, hover, focus-visible (клавиатура даёт кольцо, мышь — нет), focus, active/pressed, disabled, loading, selected/checked, error/invalid, read-only. Hover проверять **эмпирично** (реальное наведение + дифф computed-стилей), не доверять `:hover` в стайлшите. На тач — нет «залипшего» hover. Focus-ring не обрезается `overflow:hidden`. Disabled не кликается/не сабмитит по Enter, курсор `not-allowed`. Loading: спиннер в кнопке, блок повторного клика, габариты не «прыгают». Комбинации (hover+focus, disabled+loading, error+focus) предсказуемы. Состояния внутри порталов/оверлеев работают так же.
- **Сверка с макетом (Figma):** тексты символ-в-символ (регистр, пунктуация, кавычки-ёлочки, NBSP, многоточие-символ); иконки (набор/толщина/размер/цвет/filled-outline); размеры/отступы/padding/gap/кликабельная зона — численно (DevTools), не на глаз; типографика (family/size/weight/line-height/letter-spacing/цвет) по токену; выравнивание (baseline иконки+текста, сетка); радиусы/border/тени численно. **Расхождения → вопрос аналитику/дизайнеру.**
- **Консистентность токенов:** один «чёрный»/«акцентный» между экранами (нет #000 vs #0A0A0A); единые радиусы/тени/шрифт/шкала размеров; сетка отступов 4/8-pt (нет 7/13/18px); одинаковые габариты однотипных контролов; единый hover/focus/disabled-паттерн.
- **Overflow/переносы:** длинный текст в кнопке/чипе/заголовке (перенос vs ellipsis по дизайну); длинное слово/URL не вылезает (`word-break`); line-clamp обрезает корректно; плейсхолдер длиннее поля; длинные значения в таблицах (тултип); динамический контент (пустой/длинный/эмодзи/RTL); большие числа (999+, 1 000 000); текст при zoom 200%.
- **Z-index/порталы/оверлеи:** модалка/дропдаун/тултип/тост поверх всего и sticky-хедера; корректный stacking нескольких оверлеев; бэкдроп перехватывает клики, scroll-lock без «прыжка»; закрытие по бэкдропу/Esc/крестику + возврат фокуса на триггер; дропдаун у края (flip/shift), не обрезается в `overflow:hidden`/`transform`-родителе.
- **Транзиентные/loading:** мокать задержку (route/throttling) и снять состояние — спиннеры, скелетоны, прогресс, disabled-кнопки; скелетоны по геометрии = контент (нет скачка layout); нет мелькания «пусто→скелетон→пусто→данные»; оптимистичные апдейты с откатом при ошибке; долгий запрос (прогресс/отмена/таймаут вместо вечного спиннера); параллельные загрузки не дёргают layout.
- **Анимации:** длительность/easing, нет дёрганья и «прыжка»; не блокирует взаимодействие; симметрия открытия/закрытия без обрезки контента; **`prefers-reduced-motion` отключает/упрощает** (часто забывают); нет бесконечных анимаций вне видимости; резерв места против CLS.
- **Тёмная тема:** нет белых вспышек (фон/инпуты/оверлеи/iframe), нет чёрного на тёмном; иконки/SVG `currentColor` перекрашиваются; тени заменены на border/подсветку; состояния различимы; переключение на лету без вспышки и перезагрузки, persist, уважение `prefers-color-scheme`.
- **Пустые состояния/тосты:** empty (иллюстрация + текст + CTA), отличается от error и loading; «нет данных» vs «ничего не найдено по фильтру» — разные тексты/действия; тосты (позиция, автозакрытие, ручное закрытие, стэкинг, не перекрывают важное, `role=status/alert`); тип тоста различим не только цветом; inline vs тост vs модалка по назначению.
- **Инструментальные проверки:** снимать computed-стили через `browser_evaluate`/DevTools для точного диффа (не скриншот на глаз); скриншот-снапшоты ключевых состояний с мок-данными (маскировать динамику); рендер при недогруженных шрифтах (FOUT/FOIT) и заблокированных картинках (alt/плейсхолдеры); консоль без ошибок рендера (React keys, hydration mismatch), без 404 на шрифты/иконки.
- **Драйвинг контролируемых инпутов (React/Vue):** значение — через нативный setter + dispatch событий input/change, иначе стейт фреймворка не обновится; маскированные поля печатать посимвольно, не вставкой; после ре-рендеров ссылки/id элементов слетают — переискивать элемент заново.

### 2.3 Адаптив и кросс-браузер
- **Канонические разрешения Desktop:** 1920×1080 (эталон), 1536×864 (частый ноут, scale 125%), 2560×1440 (2K — растягивание, max-width контейнера, поля по бокам); доп. 1366×768, 1440×900.
- **Канонические Mobile:** 414×896 (iPhone 11/XR), 360×800 (средний Android), 393×873 (Pixel 7), 430×926 (iPhone 14/15 Pro Max); доп. 320px (минимум, частый источник переполнения), 768×1024 (iPad, граница планшет/десктоп).
- **Breakpoints:** рендер ровно на границах (767/768, 1023/1024, 1279/1280) — нет «прыжков», обрезки, дублей меню desktop+mobile; плавный ресайз 320→1920 (момент перестроения, нет наложений/схлопывания в 0).
- **Горизонтальный скролл (критично):** на каждом разрешении нет горизонтального скролла body/контейнеров (фикс-ширины px, длинные URL, широкие таблицы, отрицательные margin, `100vw` со скроллбаром, картинки без `max-width:100%`).
- **Перестроение:** многоколоночность → одна колонка (визуальный порядок = DOM, не сломан `order`); сайдбар вниз/в бургер; меню → бургер; табы → аккордеон/скролл; скрытие подписей/колонок, замена таблицы на карточки, фильтры в боттом-шит; primary-кнопки на всю ширину; sticky-CTA не перекрывает контент/safe-area.
- **Тач:** таргеты ≥44×44 (iOS)/48×48 (Android) с отступами; свайпы/слайдеры/pull-to-refresh/long-press; momentum-скролл; hover-функционал доступен по тапу, нет залипшего hover.
- **Ориентация/клавиатура:** портрет↔ландшафт без обрезки; ландшафт на коротком экране не прячет CTA; виртуальная клавиатура не перекрывает поле (скролл к нему); `100vh` не «прыгает» (использовать `dvh`/`svh`).
- **Zoom/масштаб шрифта:** zoom 200% — одна колонка без горизонтального скролла, ничего не обрезано; 400% на 1280px; увеличенный системный шрифт не ломает; pinch-to-zoom не заблокирован (`user-scalable=no` — антипаттерн).
- **safe-area/notch:** `env(safe-area-inset-*)`, контент не под вырезом/home-indicator в портрете и ландшафте.
- **Кросс-браузер — матрица:** Chromium (Chrome/Edge), WebKit (Safari desktop + iOS), Firefox; на iOS все браузеры = WebKit. **WebKit-специфика:** `position:sticky` в таблицах, `gap` во flex, `backdrop-filter`, `100vh` (адресная строка), `date`/`time` инпуты, `border-radius`+`overflow:hidden`. **Firefox:** скроллбары, кастомные чекбоксы, стрелки number, sub-pixel шрифты, `<details>`. **iframe:** скролл/фокус на iOS, высота на мобиле, sandbox.
- **Прочее:** `mask-image`/`clip-path`/`mix-blend-mode` (fallback в WebKit/FF); WebP/AVIF fallback, `srcset`/`sizes` под DPR; FOUT/FOIT, `font-display`; sticky-хедер, scroll-snap, scroll-lock; боттом-шиты (закрытие свайпом, скролл внутреннего тела); Retina 2x/3x (без размытия); перф на мобиле (тяжёлые тени/blur, 60fps); `prefers-reduced-motion`/`prefers-color-scheme`/`prefers-contrast`/forced-colors.
